# Рефакторинг проекта Marketing Bot

## 🎯 Цели рефакторинга

Исправление критических архитектурных проблем и улучшение качества кода.

## 🔧 Исправленные проблемы

### 1. Кроссплатформенность
- **Было**: Использование `fcntl` (только Unix/Linux)
- **Стало**: Кроссплатформенная блокировка процессов (`process_lock.py`)
- **Результат**: Бот теперь работает на Windows, Linux и macOS

### 2. Дублирование функций
- **Было**: Две функции для создания OpenAI thread (`_create_or_get_thread` и `get_or_create_thread`)
- **Стало**: Единая функция в `openai_client.py`
- **Результат**: Убрана путаница и дублирование кода

### 3. Конфликт кэшей
- **Было**: Два разных кэша авторизации (`FAILED_AUTH_CACHE` и `AUTH_CACHE`)
- **Стало**: Единый кэш в `auth_cache.py`
- **Результат**: Синхронизированная работа кэшей, лучшая производительность

### 4. Централизация конфигурации
- **Было**: Хардкод URL и констант в коде
- **Стало**: Централизованная конфигурация в `config.py`
- **Результат**: Легко изменять настройки, нет дублирования

### 5. Улучшенная работа с OpenAI
- **Было**: Смешанная синхронная/асинхронная логика
- **Стало**: Асинхронный клиент с retry/backoff в `openai_client.py`
- **Результат**: Стабильная работа с API, лучшая обработка ошибок

## 📁 Новые модули

### `config.py`
- Константы для колонок Google Sheets
- Настройки авторизации и OpenAI
- URL веб-приложений
- Функции-помощники

### `auth_cache.py`
- Единый кэш авторизации
- Управление блокировками
- TTL для разных типов данных

### `openai_client.py`
- Асинхронный клиент OpenAI
- Управление thread'ами
- Retry/backoff логика

### `process_lock.py`
- Кроссплатформенная блокировка процессов
- Поддержка Windows (msvcrt) и Unix (fcntl)
- Автоматическая очистка устаревших блокировок

## 🚀 Преимущества новой архитектуры

1. **Надежность**: Кроссплатформенность, лучшая обработка ошибок
2. **Производительность**: Оптимизированные кэши, асинхронность
3. **Поддерживаемость**: Модульная структура, централизованная конфигурация
4. **Масштабируемость**: Легко добавлять новые функции
5. **Тестируемость**: Разделение ответственности между модулями

## 📋 Что изменилось в bot.py

- Удалены дублирующиеся функции
- Заменены старые кэши на новые
- Использование новых модулей вместо встроенной логики
- Упрощена функция `main()` с новым ProcessLock
- Все URL теперь берутся из конфигурации

## 🔄 Миграция

1. Установить новые зависимости: `pip install -r requirements.txt`
2. Убедиться, что все модули находятся в одной папке
3. Перезапустить бота

## ⚠️ Важные замечания

- Новые модули требуют Python 3.7+
- Добавлена зависимость `psutil` для Windows
- Конфигурация теперь централизована в `config.py`
- Логирование осталось прежним
