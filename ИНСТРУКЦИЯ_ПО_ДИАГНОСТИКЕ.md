## 🎯 ГДЕ ИСКАТЬ ДАННЫЕ АВТОРИЗАЦИИ

### Основная таблица авторизации (SHEET_URL):
- Откройте таблицу по ссылке из переменной SHEET_URL
- Найдите лист "список сотрудников для авторизации"
- Посмотрите колонки:
  - **Колонка A**: Код партнера
  - **Колонка B**: ФИО
  - **Колонка C**: Телефон  
  - **Колонка D**: Статус авторизации ← ЗДЕСЬ должно появиться "авторизован"
  - **Колонка E**: Telegram ID ← ЗДЕСЬ должен появиться ID пользователя

### Таблица обращений (TICKETS_SHEET_URL):
- Это отдельная таблица для диалогов и сообщений
- Сюда НЕ записывается статус авторизации
- Сюда записываются вопросы и ответы пользователей

## 🚀 ТЕСТ АВТОРИЗАЦИИ

1. Запустите бота: `python bot.py`
2. Откройте основную таблицу авторизации в браузере
3. Попросите пользователя авторизоваться через бота
4. Следите за изменениями в колонках D и E
5. После авторизации должно появиться:
   - В колонке D: "авторизован"  
   - В колонке E: Telegram ID пользователя

## ❓ ЕСЛИ ДАННЫЕ НЕ ПОЯВЛЯЮТСЯ

### Проверьте:

1. **Правильность таблицы**: Убедитесь что смотрите именно ту таблицу, что указана в SHEET_URL

2. **Доступы Google Sheets**: Проверьте что файл credentials.json корректный и у сервисного аккаунта есть доступ к таблице

3. **Лог бота**: При авторизации в консоли должны появиться сообщения типа:
   ```
   Authorization successful for user 123456
   Updating auth status for user 123456 in row 5
   ```

4. **Существование пользователя**: Пользователь должен быть заранее добавлен в таблицу со своим кодом и телефоном

## 🔧 КОМАНДЫ ДЛЯ ОТЛАДКИ

В боте доступны команды:
- `/check_auth` - проверка авторизации
- `/push` - очистка кэша
- `/fix_telegram_id` - исправление ID в таблице

## 💡 ВАЖНО

Если вы видите сообщения в таблице обращений, но не видите статус авторизации - это НОРМАЛЬНО! Это разные таблицы для разных целей.

Авторизация работает и записывается в основную таблицу сотрудников.
