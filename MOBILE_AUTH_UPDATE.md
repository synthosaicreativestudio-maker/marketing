# 📱 Обновление системы авторизации — Mobile-Friendly

## 🎯 Проблема
Авторизация через мини-приложение не работала стабильно на мобильных устройствах из-за ограничений WebApp API.

## ✅ Решение
Реализована гибридная система авторизации с двумя методами:

### 1. **WebApp метод** (приоритетный)
- Работает через `tg.sendData()` на совместимых устройствах
- Автоматическое закрытие WebApp после отправки данных

### 2. **Команда в чат** (резервный)
- Для мобильных устройств и случаев сбоя WebApp API
- Автоматическое копирование команды в буфер обмена
- Формат: `/auth <код_партнера> <телефон>`

## 🔧 Как это работает

### Сценарий 1: Успешная WebApp авторизация
```
Пользователь → Заполняет форму → WebApp API → Бот → Авторизация
```

### Сценарий 2: Мобильная авторизация
```
Пользователь → Заполняет форму → Команда копируется → Пользователь вставляет в чат → Бот → Авторизация
```

## 📝 Формат команды авторизации

```
/auth PARTNER123 89991234567
```

**Параметры:**
- `код_партнера`: минимум 3 символа
- `телефон`: 10 или 11 цифр (автоматически нормализуется до 11)
