# 🤖 Marketing Bot

Telegram-бот для маркетинговых задач с системой авторизации сотрудников через Google Sheets и интеграцией с Redis для управления состоянием.

## 🚀 Быстрый старт

### Установка
```bash
git clone https://github.com/your-username/marketing-bot.git
cd marketing-bot
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Настройка
1. Скопируйте `.env.example` в `.env` и заполните переменные
2. Настройте Google Sheets API и сервисный аккаунт
3. Создайте Upstash Redis базу данных
4. Создайте Telegram бота через @BotFather

### Запуск
```bash
python bot.py
```

## Функциональные Требования: Авторизация

### 1. Первое взаимодействие и запрос авторизации

*   Когда пользователь добавляет бота или запускает его впервые, бот отправляет сообщение: "Для продолжения работы, вам необходимо авторизоваться".
*   Под сообщением появляется reply-кнопка "Авторизоваться".
*   Нажатие на эту кнопку напрямую открывает веб-приложение `index.html` (мини-приложение авторизации).

### 2. Мини-приложение авторизации (`index.html`)

*   Содержит два поля для ввода данных:
    *   **Поле 1: "Код сотрудника"** (принимает только цифры).
    *   **Поле 2: "Телефон из КОСМОС"** (формат `89827701055`). Первая цифра `8` зафиксирована и не может быть изменена или удалена.
*   Под полями ввода расположена кнопка "Авторизоваться" (стандартная HTML-кнопка).

### 3. Логика авторизации

*   После того как сотрудник ввел данные и нажал кнопку "Авторизоваться", бот отправляет сообщение: "Проверяю данные".
*   Бот ищет совпадение "Кода сотрудника" и "Телефона из КОСМОС" в одной строке в Google Таблице по адресу: `https://docs.google.com/spreadsheets/d/1_SB04LMuGB7ba3aog2xxN6N3g99ZfOboT-vdWXxrh_8/edit?usp=sharing`.
*   **Если данные найдены:**
    *   Бот автоматически проставляет в **колонку D** значение "Авторизован".
    *   В **колонку E** записывается Telegram ID пользователя.
    *   В **колонку F** записывается текущая дата авторизации.
    *   Бот пишет сообщение: "Вы успешно авторизованы".
*   **Если данные не обнаружены:**
    *   Бот пишет сообщение: "Данные не обнаружены, повторите попытку авторизации".
    *   Снова появляется reply-кнопка "Авторизоваться", которая напрямую открывает `index.html`.

### 4. Поведение после авторизации

*   Если сотрудник авторизован, бот больше никогда и ни на каком экране не запрашивает авторизацию.
*   Бот дважды в сутки мониторит наличие авторизованного пользователя в таблице:
    *   В **06:00 по времени Екатеринбурга**.
    *   В **15:00 по времени Екатеринбурга**.
*   **Если пользователь по-прежнему в таблице авторизации:** Все хорошо, работа бота продолжается.
*   **Если пользователь пропал из таблицы:** Бот автоматически делает рестарт к самому началу, отправляя сообщение: "Для продолжения работы, вам необходимо авторизоваться".
