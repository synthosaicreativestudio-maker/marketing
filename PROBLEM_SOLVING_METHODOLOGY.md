**⚠️ КРИТИЧЕСКИ ВАЖНО:** Данная последовательность ОБЯЗАТЕЛЬНА для выполнения при любых изменениях в проекте. Нарушение порядка может привести к новым проблемам и нестабильности системы.

## 🔍 **ЭТАП 1: ИЗУЧЕНИЕ ПРОБЛЕМЫ**

### 📊 Что нужно сделать:
1. **Детальный анализ симптомов**
   - Что именно не работает?
   - Когда проблема возникла?
   - Какие ошибки показывает система?

2. **Изучение логов**
   ```bash
   # Проверить основные логи
   tail -50 bot.log
   
   # Поиск конкретных ошибок
   grep -i "error\|exception\|fail" bot.log
   
   # Логи системного управления
   ./manage_bot.sh logs
   ```

3. **Анализ состояния системы**
   ```bash
   # Статус бота
   ./manage_bot.sh status
   
   # Проверка процессов
   ps aux | grep python | grep bot
   
   # Диагностика системы
   python3 test_improvements.py
   ```

### 🎯 Ожидаемый результат:
- Точное понимание проблемы
- Список конкретных симптомов
- Временные рамки возникновения проблемы

## 🌐 **ЭТАП 2: ИЗУЧЕНИЕ СЕТИ И MCP**

### 🔗 Проверка сетевых подключений:
1. **Google Sheets API**
   ```bash

   python3 диагностика_авторизации.py
   python3 диагностика_акций.py
   ```

2. **Telegram API**
   ```bash
   # Проверка токена и webhook
   python3 reset_telegram_webhook.py
   ```

3. **OpenAI API**
   ```bash
   # Тестирование AI-функций
   python3 -c "from openai_client import openai_client; print('OpenAI OK' if openai_client else 'OpenAI Error')"
   ```

### 🖥️ Проверка MCP серверов:
1. **Проверка конфигурации MCP**
   ```bash
   # Проверка mcp_config.json
   cat mcp_config.json | jq .
   
   # Тестирование MCP контекста
   python3 -c "from mcp_context_v7 import mcp_context; print(f'MCP OK: {mcp_context.get_stats()}')"
   ```

2. **Проверка доступности внешних сервисов**
   - Filesystem access
   - Database connections
   - Web search capabilities
   - Git repository access

### 🎯 Ожидаемый результат:
- Статус всех внешних подключений
- Список недоступных сервисов
- Качество сетевых соединений

## 🔧 **ЭТАП 4: ИСПРАВЛЕНИЯ (ТОЛЬКО ПОСЛЕ ЭТАПОВ 1-3)**

### ⚡ Принципы безопасных изменений:
1. **Создание точки отката**
   ```bash
   # Создать backup перед изменениями
   ./create_rollback_point.sh
   ```

2. **Поэтапные изменения**
   - Одно изменение за раз
   - Тестирование после каждого изменения
   - Валидация результатов

3. **Обязательное тестирование**
   ```bash
   # После каждого изменения
   python3 test_improvements.py
   ./manage_bot.sh restart
   ./manage_bot.sh status
   ```

### 🔄 Порядок внесения изменений:
1. **Остановка сервисов**
   ```bash
   ./manage_bot.sh stop
   ```

2. **Внесение изменений**
   - Редактирование кода
   - Обновление конфигурации
   - Изменение зависимостей

3. **Валидация изменений**
   ```bash
   # Проверка синтаксиса
   python3 -m py_compile bot.py
   
   # Проверка зависимостей
   pip3 install -r requirements.txt
   ```

4. **Запуск и тестирование**
   ```bash
   ./manage_bot.sh start
   sleep 10
   ./manage_bot.sh status
   ```
