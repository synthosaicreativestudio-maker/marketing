## 🎯 **ПРОБЛЕМА**

> **"У меня опять кнопка перехода в раздел акций отправляет сообщение а не переходит в новый раздел"**

### ❌ **Что происходило:**
- Кнопка "👀 Ознакомиться подробнее" в уведомлениях о акциях
- При нажатии отправлялось сообщение "Неподдерживаемое действие"
- Мини-приложение не открывалось

---

## 🔍 **ДИАГНОСТИКА**

### 🕵️ **Найденная причина:**
Проблема была в обработчике `handle_callback_query` в файле `bot.py`:

```python
# ПРОБЛЕМНЫЙ КОД:
async def handle_callback_query(self, update: Update, context: ContextTypes.DEFAULT_TYPE):
    query = update.callback_query
    await query.answer()
    data = query.data or ''
    
    # ... обработка других callback ...
    else:
        await query.edit_message_text('Неподдерживаемое действие.')  # ❌ ПРОБЛЕМА!
```

### 🔍 **Почему это происходило:**
1. **Кнопки с `web_app`** не имеют `callback_data`
2. **Обработчик callback query** получал пустой `data`
3. **Попадал в `else` блок** и отправлял сообщение
4. **Мини-приложение не открывалось**

---

## ✅ **ИСПРАВЛЕНИЕ**

### 🔧 **Что было исправлено:**

```python
# ИСПРАВЛЕННЫЙ КОД:
async def handle_callback_query(self, update: Update, context: ContextTypes.DEFAULT_TYPE):
    query = update.callback_query
    await query.answer()
    data = query.data or ''

    # ✅ ДОБАВЛЕНО: Проверка на кнопки с web_app
    if not data:
        # Кнопки с web_app не имеют callback_data, просто отвечаем
        logger.info("🔗 Получен callback от кнопки с web_app - ничего не делаем")
        return

    # ... обработка других callback ...
    else:
        logger.warning(f"❓ Неизвестный callback data: {data}")
        await query.edit_message_text('Неподдерживаемое действие.')
```

### 📋 **Изменения:**
1. **Добавлена проверка** на пустой `data`
2. **Ранний возврат** для кнопок с `web_app`
3. **Улучшенное логирование** для отладки
4. **Сохранена обратная совместимость**
