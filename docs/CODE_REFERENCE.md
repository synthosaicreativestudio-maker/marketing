# Справочник кода

Этот документ содержит подробное описание каждого файла и функции в проекте.

## Файлы

### `bot.py`

*   **Назначение:** Основной файл запуска бота. Инициализирует Telegram `Application` и регистрирует обработчики команд и сообщений.
*   **Ключевые функции:**
    *   `main()`: Точка входа в приложение. Загружает переменные окружения, настраивает логирование, создает и запускает бота.

### `handlers/start_handler.py`

*   **Назначение:** Модуль, содержащий логику обработки команды `/start`.
*   **Ключевые функции:**
    *   `start(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None`:
        *   **Описание:** Асинхронная функция, вызываемая при получении команды `/start`. Отправляет приветственное сообщение пользователю.
        *   **Параметры:**
            *   `update` (`telegram.Update`): Объект, содержащий информацию о входящем обновлении (сообщении).
            *   `context` (`telegram.ext.ContextTypes.DEFAULT_TYPE`): Объект контекста, содержащий данные о состоянии бота.
        *   **Возвращает:** `None`.
        *   **Логика:** Извлекает имя пользователя, формирует приветственное сообщение и отправляет его.

## Файлы (продолжение)

### `docs/index.html` (Telegram Mini App)

*   **Назначение:** Веб-интерфейс для авторизации партнеров. Запускается как Telegram Mini App.
*   **Структура:**
    *   Заголовок `<h1>`: "MARKETING"
    *   Подзаголовок `<h2>`: "кабинет авторизации партнера"
    *   Поле ввода `input` (type="text"): "Код партнера" (только цифры)
    *   Поле ввода `input` (type="tel"): "Телефон партнера из КОСМОС" (с префиксом "8")
    *   Кнопка `button`: "Авторизоваться"
    *   Футер `div`: "produced by SYNTHOS AI®"
*   **JavaScript логика:**
    *   Использует `window.Telegram.WebApp` API.
    *   При нажатии кнопки "Авторизоваться":
        *   Собирает данные из полей ввода.
        *   Формирует JSON-строку с данными.
        *   Отправляет данные боту через `window.Telegram.WebApp.sendData(json_data)`.
        *   Закрывает Mini App.
