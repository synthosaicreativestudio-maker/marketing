## 📋 Процесс обновления

### 1. **При каждом изменении кода**

#### **Обязательные обновления**:
- [ ] `PROJECT_JOURNAL.md` - добавить запись о том, что сделано
- [ ] `CHANGELOG.md` - добавить изменения в соответствующий раздел
- [ ] `ISSUES_AND_IMPROVEMENTS.md` - обновить статус проблем

#### **Условные обновления**:
- [ ] `TECHNICAL_ARCHITECTURE.md` - если изменилась архитектура
- [ ] `API_REFERENCE.md` - если изменились API или методы
- [ ] `DEPLOYMENT_GUIDE.md` - если изменились требования к развертыванию
- [ ] `MAINTENANCE_GUIDE.md` - если изменились процедуры обслуживания

### 2. **При исправлении проблем**

#### **Обновить ISSUES_AND_IMPROVEMENTS.md**:
```markdown

- **Статус**: ✅ Исправлено
- **Дата исправления**: YYYY-MM-DD
- **Коммит**: commit_hash
- **Описание исправления**: что было сделано
```

#### **Обновить PROJECT_JOURNAL.md**:
```markdown
### ✅ Выполнено
#### 🐛 Исправление проблемы X
- **Проблема**: описание
- **Решение**: что было сделано

```

### 3. **При добавлении новых функций**

```markdown

- Добавить описание новой функции в соответствующий раздел
- Обновить диаграммы если необходимо
- Добавить в потоки данных если применимо

### 4. **При изменении конфигурации**

#### **Обновить DEPLOYMENT_GUIDE.md**:

## 🚀 Автоматизация

### **Скрипт обновления документации**
```bash
#!/bin/bash
# update_docs.sh

echo "Updating documentation..."

# Обновить дату в заголовках
find docs/ -name "*.md" -exec sed -i "s/Последнее обновление: .*/Последнее обновление: $(date '+%Y-%m-%d %H:%M')/" {} \;

# Проверить ссылки
markdown-link-check docs/*.md

# Проверить синтаксис
markdownlint docs/*.md

echo "Documentation updated successfully!"
```

### **Git hooks**
```bash
# .git/hooks/pre-commit
#!/bin/bash
# Проверить документацию перед коммитом
if git diff --cached --name-only | grep -q "docs/"; then
    echo "Checking documentation..."
    markdownlint docs/*.md
    if [ $? -ne 0 ]; then
        echo "Documentation has errors. Please fix before committing."
        exit 1
    fi
fi
```
