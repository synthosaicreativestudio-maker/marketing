## 📋 Структура функционала акций

### **1. Google Sheets - Источник данных**

#### **Структура таблицы "Акции":**
- **Колонка A**: Дата релиза
- **Колонка B**: Название акции
- **Колонка C**: Описание
- **Колонка D**: Статус (Ожидает/Опубликовано/Активна/Закончена)
- **Колонка E**: Дата начала
- **Колонка F**: Дата окончания
- **Колонка G**: Контент (медиа ссылки)
- **Колонка H**: Кнопка публикации
- **Колонка I**: Статус уведомления

#### **Жизненный цикл акции:**
1. **Создание** → Статус "Ожидает"
2. **Публикация** → Статус "Опубликовано"
3. **Активация** → Статус "Активна"
4. **Завершение** → Статус "Закончена"

### **2. PromotionsClient - Обработчик данных**

#### **Ключевые методы:**

##### **`get_new_published_promotions()`**
- **Назначение**: Получение новых акций для отправки уведомлений
- **Логика**: Ищет акции со статусом "Опубликовано" без отправленного уведомления
- **Возврат**: Список акций с полными данными

##### **`get_active_promotions()`**
- **Назначение**: Получение активных акций для мини-приложения
- **Логика**: Ищет акции со статусом "Активна"
- **Обработка**: 
  - Парсинг дат
  - Обработка медиа контента
  - Определение UI статуса
  - Сортировка по приоритету

##### **`mark_notification_sent(row)`**
- **Назначение**: Пометка акции как уведомление отправлено
- **Логика**: Обновляет колонку "Уведомление отправлено"

##### **`_process_media_content(content)`**
- **Назначение**: Обработка медиа ссылок
- **Поддержка**:
  - Google Drive → прямые ссылки
  - YouTube → embed ссылки
  - Прямые ссылки на изображения/видео

### **3. Bot Core - Управление процессом**

#### **Фоновые задачи:**

##### **`monitor_new_promotions()`**
- **Интервал**: Каждые 30 секунд
- **Логика**:
  1. Подключение к таблице акций
  2. Получение новых опубликованных акций
  3. Отправка уведомлений авторизованным пользователям
  4. Пометка уведомлений как отправленные

##### **`menu_command()`**
- **Назначение**: Обработка команды /menu
- **Логика**:
  1. Получение активных акций
  2. Отправка данных в мини-приложение
  3. Отображение кнопки "Акции"

#### **Обработка данных от мини-приложения:**

##### **`web_app_data()`**
- **Назначение**: Обработка данных от WebApp
- **Логика**:
  1. Валидация JSON данных
  2. Определение типа данных
  3. Направление к соответствующему обработчику

### **4. Mini App - Пользовательский интерфейс**

#### **Структура spa_menu.html:**

##### **Навигация:**
- Главное меню → "Акции и мероприятия" → "Акции"
- Специальный экран для отображения акций

##### **Функции JavaScript:**

##### **`loadPromotions()`**
- **Назначение**: Загрузка акций
- **Текущее состояние**: Использует мок-данные
- **Планируется**: Интеграция с реальными данными

##### **`renderPromotions(promotions)`**
- **Назначение**: Отображение списка акций
- **Функции**:
  - Форматирование дат
  - Определение статусов
  - Создание карточек акций
  - Обработка пустого состояния

##### **`sendToBot(data)`**
- **Назначение**: Отправка данных в бот
- **Логика**:
  1. Валидация данных
  2. JSON сериализация
  3. Отправка через Telegram WebApp API

```

```

## 🚨 Проблемы и ограничения

### **Текущие проблемы:**

1. **Мини-приложение использует мок-данные**
   - `loadPromotions()` не получает реальные данные
   - Нет интеграции с ботом для получения акций

2. **Отсутствует кэширование в мини-приложении**
   - Каждый раз загружаются все данные
   - Нет оптимизации для медленных соединений

3. **Ограниченная обработка медиа**
   - Нет предварительного просмотра
   - Нет валидации ссылок

### **Архитектурные ограничения:**

1. **Синхронные вызовы Google Sheets**
   - Блокируют event loop
   - Могут вызывать таймауты

2. **Отсутствие обработки ошибок в мини-приложении**
   - Нет fallback для недоступности данных
   - Нет retry логики
